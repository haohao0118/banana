# 猴哥传奇 · 开发文档

> 当前版本 v17 · 更新于 2026-04-27

---

## 一、项目概览

移动端优先的放置型点击游戏。点击香蕉赚钱 → 购买建筑自动产出 → 解锁科技树加成 → 探索世界树。

**技术栈**

| 项 | 选择 |
|---|---|
| 前端 | 纯 HTML + CSS + JS，无构建工具 |
| 后端 / 存储 | Supabase（账号登录 + 云端存档） |
| 渲染 | requestAnimationFrame 主循环 |

---

## 二、目录结构

```
banana/
├── index.html
├── style.css
├── gm.html              # GM 后台（独立页面）
├── DOCUMENT.md
└── js/
    ├── supabase-config.js
    ├── config.js        # 静态配置（建筑/升级数值表）
    ├── gameState.js     # 运行时状态
    ├── gameLogic.js     # 纯逻辑，不碰 DOM
    ├── ui.js            # DOM 操作，不改状态
    ├── storage.js       # 存档读写（Supabase）
    └── main.js          # 入口，初始化，事件绑定，主循环
```

脚本加载顺序不可颠倒：`supabase-config → config → gameState → gameLogic → ui → storage → main`

## 三、当前游戏系统

### 3.1 核心循环

```
点击香蕉 → 获得香蕉
建筑自动 tick → 每秒产出
购买/升级 → 加速产出
```

### 3.2 建筑系统（猴子树 Tab）

- 价格公式：`basePrice × priceGrowth ^ 已购数量`
- 每种建筑有独立 baseCPS
- 升级可叠加 building_cps_mult / synergy / grandma_variant 效果

### 3.3 科技树（升级 Tab）

- 购买条件：特定建筑数量达标 或 totalBananasEarned 门槛
- 效果类型：`cps_mult`（全局）/ `building_cps_mult`（单建筑）/ `synergy`（两建筑联动）/ `click_mult`（点击加成）
- 已购买显示绿边全彩，未购买显示灰暗

### 3.4 点击系统

- 基础值 1，click_mult 升级加倍
- 点击飘出迷你香蕉动画 + 浮动数字

### 3.5 离线收益

- 离线效率 50%，上限 2 小时
- 登录时弹窗展示离线所得

### 3.6 账号 & 存档

- Supabase 注册/登录，自动保持会话
- 云端 `game_saves` 表存档，每 30 秒自动保存
- GM 后台：`gm.html`（账号 haohao）

### 3.7 邮件系统

- 首页浮动邮件按钮，红点角标
- GM 可向玩家发送香蕉奖励邮件

### 3.8 新手引导

- 三步提示：点击 → 凑够 100 → 去买第一只猴子
- 基于 `bananaCount`（当前余额）判断，不用累计值

## 五、待规划里程碑（供选择）

### M10 · 成就系统
触发条件（总点击、总建筑、里程碑香蕉数），弹出成就卡片，给予一次性奖励加成。
可见性强，玩家有"收集感"。

### M11 · 世界树内容
当前是占位符。可做：用累计香蕉解锁世界节点 → 每个节点给全局加成或解锁新建筑类型。
属于中后期进度目标，增加游戏纵深。

### M12 · 排行榜
Supabase 查 `game_saves` 表按 `totalBananasEarned` 排序，展示前 20 名。
成本低，社交驱动力强。

### M13 · 每日任务
每天刷新 3 条任务（"点击 50 次"/"购买 2 栋建筑" 等），完成后领香蕉或科技点。
增加日活留存。

### M14 · 转生（Prestige）系统
累计香蕉达到亿级门槛后可"转生"：清空进度，换取永久倍率加成（转生币）。
解决后期无目标问题，是放置游戏的核心留存机制。

### M15 · 皮肤 & 主题
解锁不同香蕉皮肤（点击飘出样式）或界面主题色。纯 CSS filter / class 切换，实现成本低。
增加炫耀性消费动机。

### M16 · 建筑合成 / 升星
同类建筑数量达标后可"合星"，提升该建筑产量并改变外观。
参考当前科技树的 synergy 机制延伸。

### M17 · 活动 / 限时副本
节日活动香蕉（特殊产出物），限时建筑，限时任务。
需要后端配置表支持，适合稳定后再做。

### M18 · 数据看板（玩家自己）
在设置里展示产量曲线、建筑占比饼图，用 canvas 或 SVG 实现。
提升策略感，让玩家理解游戏数值。

## 六、快速调参指南

| 想改什么 | 去哪里改 |
|---|---|
| 建筑基础价格 / CPS | `config.js` → `buildings[].basePrice / baseCPS` |
| 价格增长率 | `config.js` → `buildingPriceGrowth` |
| 科技效果 | `config.js` → `upgrades[].effect` |
| 离线效率 / 上限 | `gameLogic.js` → `calculateOfflineEarnings()` |
| 点击基础值 | `config.js` → `baseClickValue` |
| 新手引导文案 | `main.js` → `updateTutorial()` |
